The position Property

THe position property specifies the type of positioning method used for an element{static relative fixed absolute,sticky

position property specifies the tyype of positioning method used for an elements There are five different position value.

  • static
  • relative
  • fixed
  • absolute
  • sticky

elements are then positioned using top, bottom, left and right. note:This properties listed will not work unless the position property is set first, they also work differently depending on the position value

Position static

elements are position static by default, static position elements are not affected by the top,botton, left and right properties. An element with position static is not positioned in any special way

Position:Relative;

An element with position relative is position relative to its initial position. setting the top, right bottom and left property of a relativly positioned element will cause it to be adjusted away from the normal position

using position relative

Position:fixed

an element with the position fixed is Position relative to the viewport which means it always stay in the same page even if the page is closed. The top right bottom and left properties are used to position the element

using position fixed

Position:Absolute

An element with position:absolute is positioned relative to the nearest position ancestor, instead of position relative to the viewport, however, If an absolute position element does not have any position ancestor, it uses the document body

note: Absolute position element are removed from the normal flow and can overlap any elements

Position:sticky

An element position:sticky is positioned based on the users sroll position, a sticky element toggles between relative and fixed, depending on the scroll position. It is position:relative untill a given offset position is met in the viewport- then it "sticks" in place like position fixed
